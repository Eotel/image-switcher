# Image Switcher

Live event image switcher with Preview/Program paradigm. Express backend + multi-page Vite frontend.

## Commands

```bash
npm run dev        # Start Express(:3456) + Vite(:5173) concurrently — open localhost:5173
npm run build      # Production build via vp build → dist/
npm run preview    # Serve production build from dist/ (NODE_ENV=production)
npm start          # Start Express only (no Vite HMR)
npm run check      # Format + lint + type check (same as `npx vp check`)
npm run licenses   # Generate third-party license info
npx vp lint --type-check --type-aware   # Full type-aware lint
npx tsc --noEmit   # TypeScript check only
```

## Architecture

```
server.js              Express API + static server (CJS)
workspace-store.js     Workspace CRUD persistence (CJS)
public/
  index.html           Control surface (main UI)
  program.html         Program output (separate window)
  about.html           About/license page
  js/
    control.ts         Control surface logic
    program.ts         Program output logic
    channel.ts         BroadcastChannel cross-window messaging
    image-utils.ts     Shared image utilities
  css/
    control.css        Control surface styles (broadcast dark theme)
    program.css        Program output styles
    about.css          About page styles
vite.config.js         Multi-page Vite config (root: public/)
```

## Key Patterns

- **Toolchain is vite-plus (`vp`)**, not raw vite — use `vp dev`, `vp build`, `vp check`, `vp lint`, `vp fmt`
- **Frontend: TypeScript, Backend: plain JS (CJS)** — only `public/js/*.ts` is type-checked
- **Dev runs two servers** — Express on :3456 (API/images), Vite on :5173 (HMR + proxy to Express)
- **Image directory structure**: `<group>/<category>/<filename>.{jpg,png,webp}` — three-level hierarchy scanned by `buildManifest()`
- **Cross-window sync** via BroadcastChannel API (`channel.ts`) — control window ↔ program window
- **Thumbnails** auto-generated by sharp into `.thumbnails/` on workspace activation

## API Endpoints

| Method   | Path                           | Description                                                     |
| -------- | ------------------------------ | --------------------------------------------------------------- |
| `GET`    | `/api/manifest`                | Current workspace image manifest (groups → categories → images) |
| `GET`    | `/api/workspaces`              | List all workspaces                                             |
| `POST`   | `/api/workspaces`              | Create workspace (`{ name, path }`)                             |
| `PUT`    | `/api/workspaces/:id`          | Update workspace                                                |
| `DELETE` | `/api/workspaces/:id`          | Delete workspace                                                |
| `POST`   | `/api/workspaces/:id/activate` | Activate workspace (builds manifest + thumbnails)               |
| `GET`    | `/api/browse?path=`            | Browse directories on the host filesystem                       |
